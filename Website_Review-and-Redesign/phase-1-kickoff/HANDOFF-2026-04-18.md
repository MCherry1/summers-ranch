## What's locked (don't re-argue these)

A full enumeration lives in the amendments file. The highest-signal decisions Matt may reference by memory:

**Structural:**
- Three-part site: public signage / public herd / admin
- Same card component public and admin via mode prop
- URLs: `/herd`, `/herd/[id]` (front), `/herd/[id]/details` (back). No separate detail page.
- Mobile: vertical swipe pages, horizontal slide transitions. Flip animation was scrapped (A1) — slide only.
- Binder organization: Bulls → Cows → Heifers → Calves → Reference

**Card system:**
- Front: 6 core stats (tag, name, sex, breed, age, status) + 4 corner affordances (DOD/SOD ribbon TL, For Sale ribbon TR, photo date BL, reg# BR)
- Back: 8 sections, sticky headers, hidden when empty ("better empty than wrong")
- Photos cycle 5s with 700ms crossfade, pause on engagement
- Photo staleness thresholds by life stage (newborn 30d → mature 365d; for-sale overrides to 60d)

**Admin & auth:**
- Own URL space at `/admin/*`. Public surfaces don't change by login state (A21).
- **Passkey auth via WebAuthn** (A22). Face ID / Touch ID. iCloud Keychain syncs. No passwords. SECRET from Marty until reveal: he doesn't know what passkeys are, he'll be delighted.
- Admin users in `data/admin-users.json`. Default: matt (owner), marty (admin), roianne (admin) (A23).
- Per-user notification preferences with SMS (Twilio, ~$15-25/year) + email (A25).
- Inquiries inbox at `/admin/inquiries/` as Phase 1 deliverable (A24). Shared pool model. Formspree scrapped.

**Branding:**
- PWA manifest with ranch's existing brand (on shirts) as home screen icon (A26). SECRET from Marty until reveal.
- Dark mode is Phase 2 priority, not indefinite defer (A27). Phase 1 lays semantic-token foundation.

**Media:**
- No watermarks (A3)
- MediaAsset gets `orientation` and `aspectRatio` fields (A5)
- Hero slots require landscape (A6)
- Static images only on herd cards — no motion (A10). Atmospheric surfaces like home hero and gallery Wall may have motion in Phase 2 (A11).
- Media pipeline: HEIC → JPEG pre-conversion, 2400px max, EXIF strip, GPS strip (A8)
- Photo naming: `images/cattle/{tag}-{name-slug}/` in Phase 1; R2 object keys by stable animalId in Phase 2 (A14)

**Style preview decisions expected from Marty/Roianne:**

- **Product instincts, not developer instincts.** He thinks in UX, operational workflows, emotional beats, long-term maintainability. He does NOT think in code syntax or implementation patterns. Keep technical explanations translated into consequences-for-users.

## Things to remember about the design target

Core design principles that reliably resolve debates in this project:

- **Better empty than wrong.** Hide rows/sections when data is null. No em-dashes, no "Not set" placeholders.
- **Stale content is worse than no content.** This is why nudges exist and why the for-sale-mature photo threshold drops to 60 days.
- **Operational innovation over presentation flash.** The site's real novelty is in architecture (iOS share sheet as primary upload, passkey auth, quality-leaderboard curation, shared admin/public card component) — not in WebGL or animations.
- **Restraint over expression.** Dignified, warm, traditional over bold, flashy, modern. Matches both the audience (ranchers) and the gift recipient (Marty).
- **The cow is the hero.** On every card, the animal owns the visual space. UI chrome stays in edge thirds.
